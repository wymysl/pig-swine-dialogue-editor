# Code & Technical Health — Hostile Critique, 2026-05-22

## Verdict
SAVE_VERSION has climbed to 21 without a single save→disk→load round-trip test; every migration test only exercises `migrate_save()` in memory, so the JSON serialization, file IO, and post-load apply path are untested for the entire history of the format. An unapproved third-party Godot addon (`addons/godot_ai/`) is committed and registers a fifth autoload `_mcp_game_helper` that AGENTS.md never sanctioned and that ships in the web export. Three core scripts (DialogueRunner 895, BattleController 723, BlueBinder 698 lines) blew past the 300-line cap that the AGENTS.md module conventions still claim is enforced.

## Findings

### F1 — No save/load round-trip coverage anywhere in the suite
**Flaw:** AGENTS.md §"Hard build invariants" mandates "a save/load round-trip against the previous sprint's fixture" for any code artifact touching state or save. Grep for `save_game(`, `load_game(`, or `set_save_path_for_tests` across `tests/`: zero call sites. The `test_save_migration_v*.gd` suite only feeds dictionaries through `Save.migrate_save(old, version)` in memory. `Save.save_game()` (`scripts/systems/save.gd:81-103`) — which calls `JSON.stringify(payload, "\t")`, opens `user://save.json`, writes bytes, closes — is never invoked by a test. Neither is `Save.load_game()` (`scripts/systems/save.gd:108-135`), which adds the parse-corrupt-file branch (`if parsed == null or not parsed is Dictionary: _fail_load(...); st.data = st.reset_state()`). The corrupt-save path silently resets progress; no test confirms `save_failed` fires, and no test confirms a round-tripped Dictionary equals its input.
**Why this fails:** Migration logic is half the contract; the disk format is the other half. A Godot 4.x point release that changes `JSON.stringify` dict-key ordering, or a future refactor that swaps `"\t"` for `2`, or a typo in `_save_path` resolution, all ship green through `test_runner.gd`. SAVE_VERSION at 21 with zero IO coverage is a one-bug-from-corruption posture.
**Remediation:** Add `tests/test_save_roundtrip.gd` that (a) instantiates `State` + `Save` standalone (mirroring the `_save()` helper in the migration tests), (b) calls `Save.set_save_path_for_tests("user://test_roundtrip.json")`, (c) populates `State.data` with a fully-flagged fixture, (d) calls `save_game()`, (e) blows away `State.data`, (f) calls `load_game()`, (g) deep-equals the result against the fixture. Add a second test asserting `save_failed.emit` fires when the file is truncated mid-JSON. Add a third asserting `load_game()` on a v7 fixture written to disk produces the same dictionary as `migrate_save(v7_dict, 7)`.
**Severity:** S1

### F2 — `addons/godot_ai/` is committed, unapproved, and autoloaded into the web export
**Flaw:** `project.godot:28` registers `_mcp_game_helper="*res://addons/godot_ai/runtime/game_helper.gd"` as a fifth autoload; `project.godot:44` enables `res://addons/godot_ai/plugin.cfg`. AGENTS.md §"Stack invariants": "No C#, no GDExtension, no third-party engine plugins without human approval." AGENTS.md §"Forbidden patterns": "Adding a runtime dependency (Godot plugin, addon) without human approval." `addons/godot_ai/runtime/game_helper.gd:1-25` extends `Node` and registers a debugger message capture (`mcp:take_screenshot`) that activates whenever the debugger channel is live; the autoload glob (`*`) means it instantiates in every exported build, web included. The runtime helper is gated by `Engine.is_editor_hint` and "silently sits idle when the debugger channel is inactive (e.g. exported release builds)", but the autoload constructor still runs on web boot, and the addon's directory ships in `res://` regardless of export-preset filters unless explicitly excluded. No SPRINT_LOG entry approves the addon; the constitution explicitly bans it.
**Why this fails:** Violates the stack invariant by name, in a single-developer project where the only person who can approve is the author. Compounds: the canonical autoload list in AGENTS.md (`State`, `Signals`, `Casebook`, `DialogueRunner`) and CLAUDE.md "Project Snapshot" now disagree with `project.godot`. A future agent reading governance will not see `_mcp_game_helper` and will not know it exists at `/root/`.
**Remediation:** Pick one. (a) Reclassify as approved tooling: add a §"Approved development addons" section to AGENTS.md listing `godot_ai`, restrict its inclusion to debug builds via `if OS.has_feature("editor") or OS.has_feature("debug"):` guards inside `game_helper.gd._ready()`, and exclude `addons/godot_ai/` from the Web export preset filter (`export_filter="exclude"` + path glob). (b) Treat as forbidden: uninstall via the editor (which strips `[autoload]` and `[editor_plugins]` entries) and add `addons/godot_ai/` to `.gitignore`. Either way, file a SPRINT_LOG entry that records the decision and update the canonical autoload list.
**Severity:** S1

### F3 — `_set_state_value` silently no-ops on unknown keys; validator catches dialogue paths only
**Flaw:** `scripts/autoload/dialogue_runner.gd:359-370` writes a dotted-path value into `State.data` and returns silently when (a) any intermediate segment is missing, or (b) the leaf key does not exist. This is the exact bug the v13 migration patched after `barista.json coffee_retry_prompt` wrote to an undeclared `chapter1.coffee_retry_decision` for an unknown number of sessions (see `save.gd:322-340`). The validator (`_validate_state` lines 510-571) now flags dialogue-declared paths against `_flatten_state_paths(State.reset_state())` — good. But `BattleController._write_chapter1_flag` (called from `battle_controller.gd:271, 280, 284, 372, 413, 424, 425, 427, 450, 453, 458, 461, 463, 465`) goes through the same silent-no-op writer with no validator gate. A typo in a `const PACKET_SLOT_STATE_KEYS` entry (`scripts/systems/battle/battle_controller.gd:145-150`) or a future flag rename writes nowhere and produces no log line.
**Why this fails:** Fail-loud (AGENTS.md Rule 12) is honored at JSON-validator boot time but violated at every non-JSON write site. The class of bug that drove the v13 migration can recur in any system that calls a path-setter, and the only signal will be a downstream trigger that mysteriously never matches.
**Remediation:** Make `_set_state_value` `push_error` on missing-path failure and return a `bool` so callers can branch. Add an opt-in `strict: bool = true` parameter (default true) that callers explicitly drop only when probing optional paths. Audit `battle_controller.gd` and `dialogue_runner.gd` for the new return-value contract. Add `tests/test_state_writer.gd` that asserts an unknown path fails loudly.
**Severity:** S1

### F4 — Three core scripts >300 lines; one is approaching 900
**Flaw:** `scripts/autoload/dialogue_runner.gd` is 895 lines, `scripts/systems/battle/battle_controller.gd` 723, `scripts/ui/blue_binder.gd` 698, `scripts/ui/dialogue_box.gd` 395, `scripts/actors/npc.gd` 309. AGENTS.md §"Module conventions" line 192: "Long scripts (>300 lines) get split." DialogueRunner now packs the catalogue loader (`_load_all_dialogues`, `_load_file`, `_merge_asia_hint_states`), the validator (`_validate_catalogue`, `_validate_state`, `_flatten_state_paths`), the trigger evaluator (`_evaluate_trigger`, `_evaluate_and_group`, `_evaluate_clause`, `_is_truthy`, `_resolve_path`, `_to_compare_str`), the mutation applier (`_apply_mutations`, `_add_argument_fragment`, `_set_state_value`, `_mark_once_seen`), the option-commit handler (`_on_dialogue_option_committed`), the dismiss handler (`_on_dialogue_dismissed`), the request resolver (`_on_dialogue_requested`, `_extract_lines`), and the character registry loader.
**Why this fails:** Single-file size is a proxy for single-file responsibility. Every one of those nine concerns reads and writes `_ctx`, `_catalogue`, and `State.data`. The constitution says split; the file says no.
**Remediation:** Split DialogueRunner into four files. (1) `scripts/autoload/dialogue_runner.gd` — keeps `_ready`, `_on_dialogue_requested`, `_on_dialogue_dismissed`, `_on_dialogue_option_committed`, owns the `ActiveStateContext`. (2) `scripts/systems/dialogue/trigger_evaluator.gd` — pure `static func evaluate(trigger: String, state_data: Dictionary) -> bool`. (3) `scripts/systems/dialogue/dialogue_catalogue.gd` — file loading + merging + the `_load_*` helpers. (4) `scripts/systems/dialogue/dialogue_validator.gd` — the validation pass, returning `Array[String]` of errors. Same surgery for BattleController (split Phase 1, Phase 2, packet writes already factored) and BlueBinder (split data binding from UI building from action handlers).
**Severity:** S2

### F5 — Smoke gate catches DialogueRunner validation errors only; other autoload boot failures pass green
**Flaw:** `tests/test_smoke.gd:33-41` checks `DialogueRunner.get_validation_errors()`, which is good. It does not inspect `Casebook` or `Save` or argument-fragment loading. `Casebook.reload()` (`scripts/autoload/casebook.gd:15-28`) `push_error`s on a missing or malformed `judgments.json` and returns an empty `_judgments` dict; smoke exits 0. `DialogueRunner._load_argument_fragments` (lines 254-270) `push_warning`s on a missing file and silently empties `_argument_fragments`, after which any `add_argument_fragment` mutation warns at runtime and the Blue Folder stays empty. None of these are surfaced by the smoke gate. The same applies to `BattleController._load_judgments / _load_opponents / load_data` failures — the controller silently sits in `_loaded = false` and any call into it returns `{}`.
**Why this fails:** "Fail loud" again. Smoke is the one place a typo in `judgments.json` is supposed to halt the build. Today it does not.
**Remediation:** Extend `test_smoke.gd` to (a) assert `Casebook.get_collected_judgments()` returns a non-empty Array OR record an explicit "smoke knows casebook is empty in chapter 1" assertion, (b) assert `BattleController._loaded == true` after Main boot, (c) attach a custom error counter via `OS.add_logger` or scan the `--log-file` for `ERROR:` / `WARNING:` lines that originate from autoload `_ready` calls and fail if any are present. The third option is cleanest because it generalizes to every future autoload.
**Severity:** S2

### F6 — 8-bucket facing math copy-pasted across three actors
**Flaw:** Identical 360°-to-cardinal discretization appears verbatim in `scripts/actors/player.gd:51-71`, `scripts/actors/npc.gd:188-218`, and `scripts/actors/asia.gd:54-75`. Each implementation hardcodes the same eight `elif` branches, the same `337.5 / 22.5 / 67.5 / ...` bucket boundaries, and the same string returns. A future change to bucket boundaries (e.g. swap to `>=292.5 → back_right`) requires three coordinated edits. A future addition of a fifth actor (the Halina patrol, or any chapter-2 NPC with movement) will copy-paste a fourth.
**Why this fails:** AGENTS.md Operating Principle 8: "Before adding code, read exports, immediate callers, shared utilities." There is no shared utility, so each new copy is the contract. AGENTS.md §"Module conventions": "One node, one responsibility" — the responsibility "angle → 8-direction string" is currently three nodes.
**Remediation:** Create `scripts/systems/facing.gd` with `static func from_vector(dir: Vector2) -> String` returning the eight canonical names. Call it from all three actors. Delete the duplicated `_update_facing` / facing-derivation blocks. Add `tests/test_facing.gd` that asserts the eight bucket boundaries and the diagonal-input case.
**Severity:** S2

### F7 — `State.data` is an untyped Dictionary; schema lives only in `reset_state()`
**Flaw:** `scripts/autoload/state.gd:12` declares `var data: Dictionary = {}`. Every consumer pays the `data.get("chapter1", {}).get(key, default)` tax and the `is Dictionary` defensive ritual (see `dialogue_runner.gd:177-209`, `battle_controller.gd:_chapter1()`, `casebook.gd:72-82`, `door.gd:122-156`, `blue_folder_pickup.gd:48-72`). The "schema" is `reset_state()` plus the migration history in `save.gd`. There is no compile-time guarantee that `chapter1.witness_cooperation` is an `int`, or that `packet_requested_remedy` is one of the four enum values, or that `routes_unlocked.residential` is even declared. The validator catches dialogue-trigger paths against the flattened dictionary; nothing checks types or enum membership.
**Why this fails:** AGENTS.md §"Module conventions": "Typed GDScript: every function signature has parameter and return types. `Variant` only when truly needed." `Dictionary` here is `Variant` with extra steps. Every silent-no-op writer bug (F3), every `state_choice` / `murrow_choice` flag added by the dialogue editor without owner code (`save.gd:372-382`), every dead-end migration that never declared an enum constraint, traces back to this.
**Remediation:** Introduce `scripts/autoload/state_schema.gd` with a `class_name Chapter1State extends RefCounted` (typed fields, enum constants for the seven string-enum flags, `to_dict()` / `from_dict(d: Dictionary)`). Migrate `State.data["chapter1"]` to a `Chapter1State` instance. Adopt one flag area at a time — start with the packet slot enums, where the cost of a typo is highest. Persistence still goes through `to_dict()` so the JSON format stays stable. Add `tests/test_chapter1_state_typed.gd` that asserts `set_packet_requested_remedy("not_a_remedy")` is rejected.
**Severity:** S2

### F8 — `door.gd` hardcodes the `/root/Main` lookup with no `class_name` for MainController
**Flaw:** `scripts/actors/door.gd:94-101` does `get_tree().get_root().get_node_or_null("Main")` then `mc.get_node_or_null("RoomTransition")`. `MainController` has no `class_name`. If `scenes/Main.tscn`'s root node is renamed (today it is `"Main"` — `scenes/Main.tscn:10`), every door's `_try_open()` `push_error`s and silently fails. There is no test that exercises a door against a renamed Main; smoke exercises one scene boot, not a transition. The same brittle pattern is replicated in any future actor that needs to reach the transition system.
**Why this fails:** Implicit string contracts ("there is a node named `Main` at root with a child named `RoomTransition`") that the type system cannot enforce. Renaming the root of `Main.tscn` is a one-line scene edit that breaks every door in the game.
**Remediation:** Add `class_name MainController` to `scripts/main_controller.gd`. Expose `static var instance: MainController` set in `_ready()`. Replace door.gd's lookup with `MainController.instance.transition.go_to(target_scene, target_spawn_id)`. Add `tests/test_door_routing.gd` that asserts `_try_open()` on a door succeeds when `MainController.instance` exists and `push_error`s loudly when it does not. Same surgery for any future actor that needs the transition.
**Severity:** S2

### F9 — Stale `.bak` / `.legacy` / `.pre_*` files in active data and scene trees
**Flaw:** Six staleness signals in committed-or-tracked space: `godot/data/dialogues/halina.json.bak`, `godot/data/asia_hints.json.bak`, `godot/data/voice_references/dialogue_samples_mr_murrow.jsonl.bak`, `..._asia.jsonl.bak`, `..._whimsy.jsonl.bak`, `..._crab.jsonl.bak`, `godot/scenes/interiors/pig_swine_office.tscn.legacy`, `godot/scenes/world/routes/office_street.tscn.pre_floor_fill`, `godot/scenes/world/routes/office_street.tscn.pre_three_band`. `pig_swine_office.tscn.legacy` is documented in CONVENTIONS.md §"Floor system"; the `.pre_floor_fill` / `.pre_three_band` and the six `.bak`s are not. `data/_drafts/` and `data/dialogues/_drafts/` contain 23 dated draft JSON files dating back to 2026-05-14. DialogueRunner's catalogue scan (`_load_all_dialogues` lines 440-465) has an explicit skip for `_rewrite` / `_v2` / `dialogues.json` filenames, but no skip for `.bak` — those files don't end with `.json`, so they're ignored, fine. The risk is the human reader: which `halina.json` is current?
**Why this fails:** AGENTS.md fixture rule ("Never modify a fixture in place; add a new one") and §Operating Principle 11 (conformance > taste) both want clean trees. The drafts directory is the right move; `.bak`s in the live tree are the wrong move.
**Remediation:** Move every `.bak` file to a `_legacy/snapshots/<date>/` directory (or delete; git already preserves history). Move `office_street.tscn.pre_*` to `_legacy/snapshots/` with a SPRINT_LOG entry recording what they preserved. Add a CI / pre-commit check that fails when a `.bak` is committed under `godot/data/` or `godot/scenes/`. Audit `data/_drafts/` — promote what's canonical, archive what's stale, delete what's superseded.
**Severity:** S3

### F10 — Dead actor scripts ship with the project and load on smoke
**Flaw:** `scripts/actors/wall_occluder.gd` (78 lines) and `scripts/actors/room_fog.gd` (85 lines) are explicitly documented as incompatible with the TileMap migration: CONVENTIONS.md §"Architectural Conflicts" line 447: "`wall_occluder.gd` is incompatible with the TileMapLayer wall topology and is not used." Grep for `WallOccluder` and `RoomFog` across scenes: zero references. The scripts still parse on boot, ship in the web export, and are loadable via `load("res://scripts/actors/wall_occluder.gd")` — a future scene author could re-instantiate the broken pattern by autocomplete.
**Why this fails:** Dead code says "still supported." `test_office_wall_visibility.gd` is documented as enforcing "no `WallOccluder` / `RoomFog` exists" in the live scene — that's a runtime assertion, not a deletion. The right answer is deletion.
**Remediation:** Move both files to `_legacy/godot_scripts/` or delete them. Update `tests/test_office_wall_visibility.gd` to additionally assert `not ResourceLoader.exists("res://scripts/actors/wall_occluder.gd")` so re-instantiation fails the next CI run.
**Severity:** S3

### F11 — Non-deterministic RNG in three player-visible paths blocks bug reproduction
**Flaw:** `scripts/actors/asia.gd:9` instantiates `RandomNumberGenerator.new()` without `randomize()` and without a seed surface. Subsequent calls (`asia.gd:23, 43`) drive Asia's patrol interval. `scripts/actors/behind_desk_zone.gd:85` uses bare `randi() % LINES.size()`. `scripts/autoload/dialogue_runner.gd:721` uses bare `randi() % idle.size()` for `idle_flavor` selection. None of these go through a centralized, seedable RNG. A playtester reporting "Asia patrolled into the cabinet at the worst possible moment during the Halina meeting" cannot be reproduced; a contributor cannot lock the seed for a regression test.
**Why this fails:** Reproducibility is a property tests need, not a property production code needs — but the absence here means no test can ever cover these code paths deterministically. The dialogue idle-flavor selection in particular is in the load-bearing dialogue path.
**Remediation:** Add `scripts/autoload/rng.gd` with `var session_seed: int` initialized from `OS.get_unix_time()` at boot, and `func randi_for(stream: String) -> int` that derives a sub-stream per consumer (Asia patrol, behind-desk ambient, dialogue idle). Expose `set_seed(seed: int)` for tests. Replace the three call sites. Save `session_seed` into `State.data.debug` so a bug report includes the seed. Cost: small. Payoff: reproducible playtests.
**Severity:** S3

## Standing concerns
The `tools/hooks/pre-commit` hook is shipped in the tree, but its installation is a manual `make install-hooks` step. A new clone (or a fresh machine for the existing developer) commits straight past `voice_audit.py` and `verify_dialogue_roundtrip.js` until someone remembers. There is no `.github/` directory, no CI workflow, no protection against the gate being skipped. Either move to a CI-enforced check or accept the project ships on developer memory.

Path-as-string contracts ride throughout the codebase — `chapter1.<flag>` dotted strings flow through dialogue triggers, on_dismiss writes, options write_path, battle_controller `_write_chapter1_flag`, casebook conditions. The validator covers some of those callsites; F3 and F7 collectively cover the rest. Until the schema is typed, the validator coverage is the floor and every new system that writes to State is a new opportunity to bypass it.
