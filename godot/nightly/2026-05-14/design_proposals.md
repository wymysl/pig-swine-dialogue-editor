## Proposal 1: Beat 13 Post-Court Reactions

**Type:** DIALOGUE  
**Addresses:** `story.txt` §Beat 13 (Return to Pig & Swine — celebration, ledger
update, fee reference), Chapter 1 emotional payoff  
**PROPOSALS.md status:** Not mentioned

### Gap

`story.txt` Beat 13 specifies named dialogue beats for five characters:
- **Pig** — celebration (fee + rent + printer + Swine retainer) plus maritime cadence; `client_fee_collected` flag set
- **Murrow** — silent ledger update (visual scripting) *and* a dry one-liner to Cula acknowledging the win and bridging to the next case
- **Crab** — pretends not to be impressed; opportunist register
- **Whimsy** — proposes turning the judgment into a musical; theatrical-distancing
- **Asia** — `asia_hint_states_ch1.json` hint 11 covers this beat (`won_court && !received_swine_postcard`)

Asia is covered. All four other recurring NPCs have **zero** `won_court == true`
states. This is the chapter's emotional payoff beat. A player who walks back to
the office after winning court currently sees every NPC fall through to their
generic pre-court `after_engagement` or `idle_flavor` lines — Crab repeats
"decorative gaps," Pig loops "the ship is not actively sinking," Murrow offers
"the paperwork is current."

### Proposed Addition

Add one new state to each of the four files. All trigger on
`chapter1.won_court == true`. Pig's state additionally sets
`chapter1.client_fee_collected` in `on_dismiss` per Beat 13's quest flag.
Murrow's state is gated with `!chapter1.complete` to avoid re-firing after the
Swine postcard stinger.

**`pig.json`** — new state `pig_court_win_celebration`, trigger
`chapter1.won_court == true`:
```
"Dr. A. Cula! We have survived another impossible thing."
"The rent will breathe. The Sikorska fee came through. The printer can wait."
"The client — yes, the client — also survived. The firm will see another month."
[beat pause — single short line]
"Mr. Swine's retainer is still — somewhere. Sea of Japan. Presumably alive."
```
Set `chapter1.client_fee_collected = true` on_dismiss.

**`murrow.json`** — new state `murrow_post_court_ledger`, trigger
`chapter1.won_court == true && !chapter1.complete`:
```
"Cula. The motion is on the record. The ledger is current. The next file is
already on the shelf."
```
One line. Dry-collegial. Then Murrow turns back to desk. No celebration.

**`crab.json`** — new state `crab_court_win`, trigger
`chapter1.won_court == true`:
```
"It was the only possible result from those facts. Good facts, correctly
presented."
```
One line. The "correctly presented" is the concession. Crab addresses no one.

**`whimsy.json`** — new state `whimsy_court_win`, trigger
`chapter1.won_court == true`:
```
"Cula. The judgment has a throughline, a reversal, and a satisfying last line.
I want the dramatic rights."
```
One line. Theatrical-distancing dressed as earnest request.

A fuller draft of the Pig state with trigger conditions, `on_dismiss` mutations,
and voice-reference citations is in:
`godot/data/_drafts/nightly_design_pig_2026-05-14.json`

## Proposal 3: Author Court Round 2 and Round 3 Data Files

**Type:** DATA  
**Addresses:** `story.txt` Beat 12 (court rounds 2–3), PROPOSALS.md §10 (two-
phase court round structure, status "DONE"), PLAN.md §Vertical slice plan step 4
(Casebook Battle System v1)  
**PROPOSALS.md status:** §10 marked DONE, but `data/court_rounds/_schema.md` is
currently marked REVERTED and carries a `git rm` instruction; the schema's open
question was never formally answered

### Gap

`data/court_rounds/ch1_round1_halina_examination.json` exists and is complete
(Phase 1 fact-finding with four press options, cooperation budget, fact-flags,
and thresholds). Rounds 2 and 3 have no data files. `battle_controller.gd`
(Session 33) loads round data from this directory; without rounds 2 and 3 the
Casebook battle loop cannot run past round 1.

The `_schema.md` was marked REVERTED due to an open design question in
`PROPOSAL_court_rounds_schema.md`: *"how does the new Phase 1 fact-flag system
layer with the existing stance-keyed bonus-evidence branches in
`judge_district_ch1.json`?"*

Inspecting `ch1_round1_halina_examination.json` and `judge_district_ch1.json`
together answers the question: the two systems are **additive and non-competing**.
`judge_district_ch1.json` gates judge *reactions* on `casebook_judge_state` +
`client_meeting_stance`. The round files gate Phase 2 *citation availability*
on Phase 1 `fact_flags`. The stance already controls which bonus evidence Halina
offered during Beat 8; those items surface as `press_option` outcomes in Phase 1
which produce fact-flags; those fact-flags gate what Cula can cite in Phase 2
closing. No overlap, no conflict.

This answer should close the REVERTED status, unblock the schema promotion, and
allow rounds 2 and 3 to be authored to the same pattern as round 1.

### Proposed Addition

**Step 1 (resolves schema REVERTED):** Confirm the additive-layering answer
above in `PROPOSALS.md` §10 — add one sentence to the notes field: *"Open
question resolved: Phase 1 fact-flags and stance-keyed judge reactions are
additive; fact-flags gate Phase 2 citations only."* Then restore `_schema.md`
from the pre-REVERTED content (commit c83feaa) or write a fresh one following
the round-1 pattern.

**Step 2 (Design-role deliverable):** Author two new files:
- `data/court_rounds/ch1_round2_fair_hearing.json`
- `data/court_rounds/ch1_round3_remedy.json`

Round 2 spec (from `story.txt` §Round 2):
- Phase 1: Whimsy's fair-hearing framing. Press options: invoke Rights Memo,
  invoke third-clause argument (adult occupant did not cure service), invoke
  Article 135-bis § 2 second clause.

  6 doesn't apply — these aren't criminal proceedings"; available citations gated

  `ch1_135bis_s2_understood` (set in Beat 9 archive research).
