## Task M1 — Replace placeholder barista portraits with real illustrations

**Why:** The five PNGs at `godot/art/portraits/barista/` are geometric
placeholders generated by `tools/generate_coffee_placeholders.py` using
Python's stdlib (3 KB each, flat-color shapes). Antigravity's Session 16
sprint log notes: *"Full gouache-register barista portraits were generated
via AI image tool (visible in conversation) but could not be copied from
brain directory to project due to macOS sandbox permissions."* The real
portraits never landed.

**Effort:** ~20 minutes once you have an image-gen workflow.

**Steps:**

1. Open your image generation tool of choice (Midjourney, DALL-E, Stable
   Diffusion via Automatic1111, etc.).
2. For each of the five emotions below, generate a 512×512 portrait of
   the Barista using this prompt anchor (from `godot/CONVENTIONS.md`
   §Portrait generation rules — warm register):

   > "Small format gouache painting, naive Polish illustration technique,
   > flat warm earth-tone colors, slightly satirical, expressive but
   > grounded, institutional café setting, visible brushstroke texture,
   > Polish barista woman, café apron, dignified but slightly tired
   > expression."

   Negative anchor:
   > "photorealistic, 3D render, anime, digital art, smooth gradients,
   > horror, dark, cute, big head, chibi"

   Per-emotion variation (append to the positive prompt):

   | Emotion | Filename | Expression cue |
   |---|---|---|
   | Perfect | `perfect.png` | proud, faintly approving smile |
   | Good | `good.png` | mild approval, slight nod |
   | Okay | `okay.png` | neutral, professional |
   | Bad | `bad.png` | weary, faintly disappointed |
   | Machine objects | `machine_objects.png` | resignation, hands raised at the machine |

3. Save each as 512×512 PNG. Drop them into
   `godot/art/portraits/barista/` overwriting the placeholders. Keep
   exact filenames — the wiring in `coffee_brewing.gd` reads these
   paths.

4. After replacing, open the project in Godot once so `.import` files
   get regenerated. Or run:
   ```
   godot --headless --path godot --import --quit
   ```
   from the repo root.

5. Playtest the minigame to verify the new portraits appear on the
   result screen.

**Skip if:** You don't have image generation tooling handy. The
placeholders display correctly — they just look bad. The wiring will
pick up real portraits the moment you drop them in, no further code
change needed.

## Task M4 — Real pixel art for the 24 placeholder sprites (deferred)

**Why:** The minigame's 24 sprites (machine states, cup states, prompt
icons, meters, stamps, etc.) are programmatic geometric shapes generated
by `tools/generate_coffee_placeholders.py`. They function, but they read
as crude rectangles and circles rather than pixel art.

**Effort:** ~half a day with Pixellab once the workflow is set up.
Comparable to the original character sprite generation pass.

**Why deferred:** The minigame is now mechanically and visually wired.
Real pixel art is polish, not unblocker. Better done in a dedicated
session with all 24 sprites together so the visual style stays
consistent — interleaving with code work risks one-off prompts that
drift in style.

**When to do this:** After the test-authoring and UX-legibility passes
land. At that point the minigame is "complete" in the design sense, and
the pixel-art swap is straightforward — replace each PNG at the same
path, run import.

**Skip if:** You're OK with the geometric placeholders for the
playable-prototype stage. Stylistically the placeholders match the
flat institutional vibe of the rest of the game; they're not so far
off-tone that they actively hurt.
